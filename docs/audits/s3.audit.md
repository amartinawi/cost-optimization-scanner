## Closeout

The S3 adapter section is **complete**. All 12 v1 findings resolved with the dedicated `S3_SAVINGS_FACTORS` constant (grounded in AWS S3 docs), a new bucket-error router (`_route_bucket_error`), a split renderer pair, and 26 new tests covering factor completeness, opportunity classification, AccessDenied routing, dedicated-category dedup, and the no-arbitrary-40%-fallback regression.

### Shared infrastructure added during this round (reusable by future audits)

- `services/s3.py:_route_bucket_error(ctx, bucket, exc, *, action, expected_codes)` — drop-in helper for any adapter that swallows expected misses (e.g., `NoSuchBucketPolicy`) while routing permission denials through `ctx.permission_issue`.
- `services/s3.py:_is_access_denied(exc)` — classifier covering `AccessDenied`, `AllAccessDisabled`, `AuthorizationError`, `Forbidden`, `MethodNotAllowed` plus message-substring fallback.
- `services/s3.py:_classify_opportunities(bucket_info)` — opportunity-tag → savings-factor key resolver; pattern reusable for any service with discrete savings-class categories.
- `reporter_phase_b.py:_render_s3_enhanced_checks` — config-pattern renderer template (BucketName + CheckCategory + parseable EstimatedSavings string).
